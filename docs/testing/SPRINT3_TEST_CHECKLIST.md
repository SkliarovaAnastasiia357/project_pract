# Чек-лист тестирования спринта 3

## Профиль

- [ ] Редактирование описания
  Шаги: открыть `/profile`, ввести новый текст в поле "О себе", нажать "Сохранить описание".
  Ожидаемый результат: описание сохраняется без перезагрузки, показывается сообщение об успехе, обновлённый текст виден в профиле и в боковой карточке.

- [ ] Сохранение пустого описания
  Шаги: очистить поле "О себе", сохранить форму.
  Ожидаемый результат: пустое описание допускается, ошибка не показывается, в профиле отображается состояние "Описание пока не заполнено.".

- [ ] Граница длины описания
  Шаги: сохранить описание длиной 400 символов, затем попробовать 401 символ.
  Ожидаемый результат: 400 символов сохраняются успешно, при 401 символе показывается ошибка "Описание не должно превышать 400 символов".

- [ ] Добавление навыка
  Шаги: ввести новый навык в поле "Новый навык", нажать "Добавить".
  Ожидаемый результат: навык появляется в списке сразу, без перезагрузки страницы, показывается сообщение "Навык добавлен.".

- [ ] Добавление пустого навыка
  Шаги: ввести только пробелы в поле навыка и отправить форму.
  Ожидаемый результат: отображается ошибка "Введите навык", навык не создаётся.

- [ ] Удаление навыка
  Шаги: нажать кнопку удаления у существующего навыка.
  Ожидаемый результат: выбранный навык исчезает из списка, остальные навыки остаются, показывается сообщение об успешном удалении.

- [ ] Удаление последнего навыка
  Шаги: оставить в профиле один навык и удалить его.
  Ожидаемый результат: список навыков становится пустым, вместо него показывается empty-state без ошибок интерфейса.

## Проекты

- [ ] Создание проекта
  Шаги: открыть `/projects/new`, заполнить форму валидными данными, нажать "Создать проект".
  Ожидаемый результат: происходит переход на `/home`, показывается flash об успешном создании, проект появляется в списке.

- [ ] Редактирование проекта
  Шаги: открыть `/projects/:id/edit`, изменить данные проекта, сохранить.
  Ожидаемый результат: происходит возврат на `/home`, изменения видны в карточке проекта и в списке.

- [ ] Удаление проекта
  Шаги: на главной странице нажать "Удалить", затем "Подтвердить".
  Ожидаемый результат: проект удаляется из списка без перезагрузки, карточка исчезает.

- [ ] Обязательные поля
  Шаги: попробовать сохранить проект с пустыми `title` и `description`.
  Ожидаемый результат: форма показывает ошибки "Название проекта обязательно" и "Опишите идею проекта", проект не создаётся.

- [ ] Максимальная длина полей
  Шаги: проверить значения на границе и выше границы.
  Ожидаемый результат: `title` до 80 символов включительно валиден, `description` до 500, `stack` до 160, `roles` до 160; при превышении показываются соответствующие ошибки валидации.

## Регресс

- [ ] Сквозной сценарий спринта 3
  Шаги: зарегистрироваться, заполнить профиль, добавить навык, создать проект, отредактировать проект, удалить проект.
  Ожидаемый результат: вся цепочка выполняется без поломки состояния, профиль остаётся заполненным, проект после удаления исчезает из списка.

- [ ] Регрессия существующих сценариев
  Шаги: запустить `npm test`.
  Ожидаемый результат: frontend-тесты проходят, backend-тесты проходят или корректно пропускают container-based suites в среде без Docker.
