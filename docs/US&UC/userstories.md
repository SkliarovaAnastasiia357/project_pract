# 1.	Введение и контекст

## 1.1 Цель документа

Данный документ содержит набор user stories и примеры use cases для минимально жизнеспособного продукта (MVP) платформы по подбору участников и формированию команд. 

## 1.2 Определения и сокращения
Определение	Значение
MVP, Minimum Viable Product	минимально жизнеспособный продукт
UI, User Interface	пользовательский интерфейс
API, Application Programming Interface	программный интерфейс для взаимодействия
JWT, JSON Web Token	механизм аутентификации
CI/CD, Continuous Integration / Continuous Delivery	автоматизация сборки и развёртывания
Таблица 1 – Определения, сокращения и значения.

 
# 2.	User Stories

```
1.	Как новый пользователь, я хочу зарегистрироваться, указав email, имя и пароль, чтобы получить доступ к платформе.

Подробные критерии: 
- Форма регистрации содержит поля: Email, Имя, Пароль, Подтверждение пароля.
-  После успешной регистрации пользователь автоматически входит в систему (либо перенаправляется на страницу входа с сообщением об успехе).
-  Нельзя зарегистрироваться с уже существующим email → сообщение об ошибке. 
```

```
2.	Как новый пользователь, я хочу видеть понятные ошибки валидации, чтобы  самостоятельно исправить некорректные данные.

Подробные критерии: 
- Ошибка при некорректном формате email.
- Ошибка, если пароль короче 6 символов (или заданного минимального значения).
- Ошибка, если пароль и подтверждение пароля не совпадают.
```
-------------------------------------------
```
3.	Как зарегистрированный пользователь, я хочу войти в систему, указав email и пароль, чтобы получить доступ к защищённым страницам приложения.

Подробные критерии: 
-  Форма входа содержит поля: Email, Пароль.
-  При верных данных — перенаправление на главную страницу приложения.
-  При неверных данных — общая ошибка «Неверный email или пароль» (без уточнения).
```
-------------------------------------------
```
4.	Как авторизованный пользователь, я хочу нажать кнопку «Выйти» в любом месте приложения, чтобы завершить сеанс и обезопасить свой аккаунт.

Подробные критерии:
- После выхода все защищённые страницы становятся недоступны.
- При попытке перейти на защищённую страницу после выхода - редирект на страницу входа.
```
-------------------------------------------

5.	Как авторизованный пользователь, я хочу видеть навигационное меню с пунктами «Главная», «Мой профиль», «Выйти», чтобы понимать структуру приложения и перемещаться между страницами.

Подробные критерии:
- Меню отображается на всех защищённых страницах.
- Текущий пункт меню визуально выделен.

Подробные критерии:
- Страница не падает с ошибкой, содержит информативный placeholder.

-------------------------------------------

7.	Как авторизованный пользователь, я хочу перейти на страницу «Мой профиль» и увидеть свои данные (имя, email),чтобы убедиться, что система загрузила правильную учётную запись.

Подробные критерии:
- Данные подтягиваются из текущей сессии / API текущего пользователя.
- Редактирование профиля отсутствует (допустимо серое поле/пометка «будет в следующей версии»).

8.	Как разработчик, я хочу иметь модель User с полями id, email, name, hashed_password, created_at, чтобы данные пользователей структурированно хранились в базе.

9.	Как авторизованный пользователь, я хочу добавить текстовое описание в свой профиль, чтобы другие пользователи понимали мой опыт и цели в проектах.

Подробные критерии:
- На странице профиля есть поле для ввода описания.
- Описание можно сохранить, нажав кнопку «Сохранить».
- После сохранения описание отображается в профиле.
- Максимальная длина — 1000 символов.
- Пустое описание допускается.
```
-------------------------------------------
```
10.	Как авторизованный пользователь, я хочу выбрать навыки из списка (Python, SQL, Figma и др.) и добавить их в свой профиль, чтобы меня могли найти капитаны, которым нужны эти технологии.

Подробные критерии:
- На странице профиля отображается список всех доступных навыков.
- Навыки представлены в виде чекбоксов.
- Пользователь может выбрать любое количество навыков.
- Выбранные навыки сохраняются при нажатии «Сохранить».
- После сохранения выбранные навыки отображаются в профиле как теги.
```
-------------------------------------------
```
11.	Как авторизованный пользователь, я хочу убрать навык из своего профиля (снять чекбокс), чтобы мои устаревшие или неактуальные компетенции не отображались и не мешали поиску.

Подробные критерии:
- На странице профиля напротив каждого выбранного навыка можно снять выделение.
- После снятия чекбокса и нажатия «Сохранить» навык удаляется из профиля.
- Удалённый навык перестаёт отображаться в публичном профиле.
- Удалённый навык больше не участвует в поиске по этому пользователю.
```
-------------------------------------------
```
12.	Как авторизованный пользователь, я хочу открыть профиль другого участника или капитана, чтобы перед отправкой заявки или приглашения оценить его навыки и опыт.

Подробные критерии:
- По клику на имя пользователя открывается его публичный профиль.
- В публичном профиле отображаются: имя, описание, список навыков (теги), роль.
- Кнопки редактирования отсутствуют.
- Если у пользователя нет описания — отображается заглушка «Пользователь пока ничего не рассказал о себе».
```
-------------------------------------------
```
13.	Как авторизованный пользователь с ролью Капитан, я хочу создать карточку проекта с указанием названия, идеи и технологического стека, чтобы начать поиск участников в команду.

Подробные критерии:
- На отдельной странице есть форма создания проекта.
- Поля формы: название (обязательное, макс. 100 символов), идея/описание (обязательное, макс. 2000 символов), технологический стек (текстовое поле).
- После сохранения проект появляется в списке проектов.
- Капитан автоматически становится владельцем проекта.
- Созданный проект получает статус «активен».
```
-------------------------------------------
```
14.	Как авторизованный пользователь с ролью Капитан (владелец проекта), я хочу изменить название, описание или технологический стек моего проекта, чтобы корректировать информацию по мере развития проекта.

Подробные критерии:
- На странице проекта есть кнопка «Редактировать» (видна только капитану-владельцу).
- При нажатии открывается форма, предзаполненная текущими данными.
- Можно изменить любое поле.
- После сохранения изменения отображаются на странице проекта.
- Нельзя редактировать чужие проекты (кнопка отсутствует или доступ запрещён).
```
-------------------------------------------
```
15.	Как авторизованный пользователь, я хочу открыть страницу любого проекта и увидеть его название, описание, технологический стек и владельца, чтобы понять, подходит ли мне этот проект для участия.

Подробные критерии:
- По клику на название проекта (в списке) открывается его карточка.
- На карточке отображаются: название, идея/описание, технологический стек, владелец (имя капитана), статус (активен/закрыт).
- Если проект закрыт — это явно указано.
- Редактирование недоступно для не-владельцев.
```
-------------------------------------------
```
16.	Как авторизованный пользователь с ролью Капитан, я хочу видеть список всех проектов, которые я создал, чтобы быстро управлять ими и отслеживать их статус.

Подробные критерии:
- В личном кабинете или отдельном разделе есть вкладка «Мои проекты».
- Отображаются только проекты, где текущий пользователь — владелец.
- Для каждого проекта видно: название, статус (активен/закрыт), дату создания.
- По клику на название проекта открывается его карточка для просмотра/редактирования.
```
-------------------------------------------
```
17.	Как авторизованный пользователь, я хочу видеть список всех активных проектов на платформе, чтобы выбирать, в каком проекте я хотел бы участвовать.
Подробные критерии:
- На главной странице или в разделе «Проекты» отображается список проектов.
- Показываются только проекты со статусом «активен».
- Для каждого проекта видно: название, краткое описание (первые 100 символов), владельца.
- По клику на название открывается полная карточка проекта.
```
-------------------------------------------
```
18.	Как пользователь платформы, я хочу ввести ключевое слово или выбрать навык и получить список проектов и людей, строго соответствующих критериям, чтобы быстро найти релевантную команду или участника.

Подробные критерии:
- Единая поисковая строка в шапке сайта.
- Фильтр по навыкам (выпадающий список или чекбоксы) — строгое совпадение: например, при выборе «Python+SQL» показываются только те, у кого оба навыка.- Если проект закрыт — это явно указано.
- Поиск по ключевым словам ищет в названии/идее проекта и в описании профиля.
- Результаты выдаются в виде карточек с краткой информацией и ссылкой на детальную страницу.

```
19.  Как пользователь, я хочу отправить заявку на указанную в проекте роль, чтобы предложить свою кандидатуру капитану.

Подробные критерии:
- На странице проекта рядом с каждой ролью есть кнопка типа «Подать заявку».
- При нажатии открывается форма (или сразу отправляется заявка с комментарием, если есть).
- Пользователь не может отправить больше одной заявки на одну и ту же роль в проекте.
- Капитан получает уведомление (в интерфейсе, в разделе  типа «Входящие заявки»).
```
-------------------------------------------
```
20.     Как капитан проекта, я хочу видеть список входящих заявок на роли и принимать/отклонять их, чтобы формировать команду из подходящих участников.

Подробные критерии:
- Владелец проекта имеет раздел «Заявки на мой проект» с фильтром по ролям.
- Заявка показывает: пользователя, его навыки (теги из профиля), роль, на которую претендует, дату подачи.
- Кнопки «Принять» и «Отклонить».
- После принятия пользователь уведомляется (в интерфейсе), статус заявки меняется на «Принята».
- После отклонения — статус «Отклонена», пользователь может подать заявку повторно (если капитан разрешит).
```
